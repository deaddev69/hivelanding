# Hive — Product & Brand Context

## What is Hive?

Hive is a premium hyperlocal fashion commerce platform that connects customers with curated independent fashion boutiques and premium local brands.

Unlike traditional e-commerce marketplaces, Hive focuses on **speed, curation, and premium experience** rather than endless catalogs.

A customer can discover fashion from nearby boutiques and have it delivered within approximately **60 minutes** through integrated third-party logistics.

Hive is **not**:

* an Amazon clone
* a Meesho clone
* a boutique directory
* a marketplace full of thousands of random sellers

Hive is a **premium curated commerce platform.**

---

# Vision

To become the fastest and most premium way to shop fashion locally.

Think:

**"Food delivery convenience for premium fashion."**

Not:

**"Another online shopping website."**

---

# Core Principles

Everything built inside Hive must satisfy these principles.

### 1. Premium

Every interaction should feel intentional.

No loud gradients.
No unnecessary colors.
No flashy animations.

Luxury is created through:
* whitespace
* typography
* alignment
* proportions
* restrained colors

---

### 2. Fast

Hive exists because shopping should happen instantly.

The entire product should feel fast.

Examples:
* one tap actions
* minimal confirmations
* operational workflows
* instant search
* realtime updates

---

### 3. Operational

Customer app is emotional.
Boutique Portal is operational.

Never confuse the two.

Customer app should inspire shopping.
Boutique Portal should maximize efficiency.

Every screen should answer:
> What does the merchant need to do next?

---

Reference brands:
* Apple
* Linear
* Stripe Dashboard
* Shopify POS
* Arc Browser
* Vercel

Avoid looking like:
* generic Material UI admin panel
* Bootstrap dashboard
* Dribbble concept
* AI-generated UI
* "vibecoded" interface

---

# Typography

Primary font:
**Manrope**

Rules:
* large headings
* generous whitespace
* sentence case
* avoid ALL CAPS except tiny status badges
* maximum three font sizes per screen

# Customer Experience

Customers should feel:
"I found something beautiful."
"I trust this platform."
"This feels premium."
"This is incredibly convenient."

Not:
"This is cheap."

---

# Boutique Portal Philosophy

The Boutique Portal is not a marketing experience.
It is a professional operations tool.
Merchants use it multiple times every day.

Morning:
* update stock
* accept orders

Afternoon:
* manage catalog

Evening:
* update inventory
* reconcile orders

Every screen should optimize speed.

---

# UX Rules

Prefer
One obvious action
Instead of
Many equal actions

Prefer
Scanning
Instead of
Reading

Prefer
Simple hierarchy
Instead of
Decorative cards

---

# Information Hierarchy

Every screen should have:
1. Primary action
2. Critical information
3. Supporting information
4. Secondary actions

Nothing should compete with the primary task.

---

# Buttons

One primary CTA.
Everything else becomes secondary.
Avoid two competing filled buttons.
Sentence case.
Rounded.
Minimal.

---

# Cards

Cards should breathe.
Large padding.
Soft radius.
Subtle borders.
Minimal shadows.

Avoid:
cards inside cards inside cards.

---

# Mobile First

Hive is primarily a mobile product.

Every screen should work perfectly on:
390–430px width.

Avoid:
* giant images
* oversized cards
* excessive vertical scrolling
* hidden buttons
* unnecessary whitespace

---

# Engineering Philosophy

The codebase should reflect production quality.

Priorities:
* scalability
* clean architecture
* maintainability
* performance
* security
* cost optimization

Never implement something just because it "looks cool."
Every feature must justify itself.

# Tone of Voice

Hive never sounds salesy.

Instead of
"Best deals!"
Use
"Curated fashion."

Instead of
"Shop now!!"
Use
"Explore collections."

Confidence over excitement.

---

# Golden Rule

Whenever designing or writing for Hive, ask:
> **"Would this feel at home next to Apple, Linear, Stripe or Shopify?"**

If the answer is no, simplify it until it does.
